### 0. Опредления
**Pod (под): базовая единица развертывания**  
- **Что это?** 
Pod — это самая маленькая и базовая единица в Kubernetes, которая представляет собой один или несколько связанных контейнеров. Все контейнеры внутри Pod делят одну сеть и одно хранилище (volume).
 
- **Основные особенности:**  
  - Каждый Pod может содержать:
    - Один контейнер (наиболее распространённый случай).

    - Несколько контейнеров, которые работают вместе (например, когда дополнительный контейнер поддерживает основной).
 
  - Контейнеры в Pod:
    - Делят один IP-адрес.

    - Общаются друг с другом через локальный хост (localhost).

  - Pod обычно живёт недолго. Если Pod выходит из строя, Kubernetes создаёт новый Pod вместо него (но с другим идентификатором).
 
- **Используется для:** 
  - Запуска контейнеров и управления их жизненным циклом.

  - Группировки контейнеров, которые работают как часть одного микросервиса.



**Узел (node): физический или виртуальный сервер**  
- **Что это?** 
Узел (node) — это физический сервер или виртуальная машина, на которой запущен Kubernetes. Узел предоставляет вычислительные ресурсы (CPU, память, сеть, хранилище) для запуска Pod.
 
- **Основные особенности:**  
  - На каждом узле запускаются следующие компоненты: 
    - **Kubelet** : агент, который управляет Pod на этом узле.
 
    - **Container runtime** : например, Docker или containerd, отвечает за запуск контейнеров.
 
    - **Kube-proxy** : обеспечивает сетевое взаимодействие между Pod и внешним миром.
 
  - Узел может быть: 
    - **Мастер-узлом (control plane node)** , который управляет кластером.
 
    - **Рабочим узлом (worker node)** , на котором запускаются Pod.

  - В кластере может быть несколько узлов, что позволяет распределять нагрузку и обеспечивать отказоустойчивость.
 
- **Используется для:** 
  - Обеспечения физической или виртуальной среды для запуска Pod.

  - Управления рабочей нагрузкой (через размещение Pod).

### 4. Создание Pod

**Задание:**  Развернуть Pod в Minikube.

1. Запускаем Minikube:

```bash
minikube start
```

2. Проверяем, что узел запущен:

```bash
kubectl get nodes
```

**Определение:**  Pod — минимальная сущность в Kubernetes, представляющая собой одну или несколько контейнеров,
работающих в одной логической среде.

1. Создаем **YAML-манифест**  для Pod:

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: vault
  labels:
    environment: dev
    tier: vault
spec:
  containers:
    - name: vault
      image: vault
      ports:
        - containerPort: 8200
```

2. Выполняем команду для создания Pod:

```bash
kubectl create -f vault_pod.yaml
```

3. Проверяем наличие Pod:

```bash
kubectl get pods
```

![Pod Vault](pics/4.png)

### 5. Создание сервиса

**Задание:**  Организовать доступ к Pod через сервис.

1. Создаем сервис:

```bash
minikube kubectl -- expose pod vault --type=NodePort --port=8200
```

2. Перенаправляем трафик на локальный порт:

```bash
minikube kubectl -- port-forward service/vault 8200:8200
```

*сервис уже заранее существовал это вторая попытка*
![Port-forward](pics/5.png)

3. Проверяем работу Vault, открыв страницу авторизации:
   `http://localhost:8200`.
